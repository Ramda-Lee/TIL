# TCP/IP 개념정리

## OSI 7계층과 TCP/IP 4계층 매핑 정리
- 응용, 표현, 세션 (TCP/IP: 응용 계층), 데이터 이동 : Data
- 전송 계층 (TCP/IP: 전송 계층), 데이터 이동 : Segment
- 네트워크 계층 (TCP/IP: 인터넷 계층), 데이터 이동 : Packet
- 데이터링크, 물리 (TCP/IP: 네트워크 엑세스 계층), 데이터 이동 : Frame, bit
## IP(목적지 주소)와 TCP(신뢰성 전송)의 역할 구분
- P는 보낼 도착 주소, TCP는 신뢰성 흐름 제어의 역할
- IP는 단위가 패킷, TCP는 segment
- 패킷 전달 방식(IP)
    - 목적지 IP 확인
    - 라우팅 테이블로 경로 결정
    - 다음 라우팅(홉)으로 전달
    - 목적지 도달
- TCP 신뢰성 보장 방식
    - 3-way Handshake 연결
    - 데이터 전송 + Seq 번호
    - 수신측 ACK 응답
    - 미수신 시 자동 재전송
## 3-way handshake 흐름 이해 (SYN → SYN+ACK → ACK)
- 3번 보내는 이유: 양방향 통신을 확인하기 위해
- Seq 번호의 의미: 이후 데이터 전송 시 패킷 순서를 추적하기 위해(없다면 재전송)
- 1단계 — SYN
Client → Server : SYN
Client가 "연결하고 싶다"는 신호를 보낸다. 이때 임의의 Sequence Number(ISN)를 생성해서 함께 전송한다. Client는 SYN_SENT 상태가 된다. (SYN = 1, Seq = 100 (임의 생성), ACK = 0)
- 2단계 — SYN+ACK
Server → Client : SYN + ACK
Server가 요청을 받고 "알겠다, 나도 연결 준비됐다"는 응답을 본다. ACK는 받은 Seq+1, 자신의 새 Seq도 함께 전송한다. Server는 SYN_RECEIVED 상태가 된다.
(SYN = 1ACK = 1Seq = 300 (Server ISN)Ack = 101 (100+1))
- 3단계 — ACK
Client → Server : ACK
Client가 Server의 SYN+ACK를 받고 최종 확인 응답을 보냅니다. 이 ACK가 도착하면 양쪽 모두 ESTABLISHED 상태가 되고, 이제 HTTP 요청 등 실제 데이터를 전송할 수 있습니다.
(ACK = 1Seq = 101Ack = 301 (300+1)이후 데이터 전송 시작)
- Tip: Springboot에서는 @RestController가 HTTP응답을 내려보내기 전에 이미 3-way HandShake가 완료된 상태이다.
## 포트와 소켓의 개념 이해
- 포트는 IP 주소 안에 어떤 프로그램과 통신할지 구분하는 논리적 번호(0 ~ 65535 사이)
- 소켓은 IP + Port의 조합으로 만들어지는 실제 통신 엔드포인트, 운영체제가 관리한다.
