# nginx 리버스 프록시 Test (Docker 버전)

## 1. 전체 아키텍처
```jsx
[ 호스트 PC (Mac) ]
│
├─ Docker Desktop
│   └─ 컨테이너 3개
│       ├─ client   (curl만 깔린 컨테이너)
│       ├─ nginx    (리버스 프록시)
│       └─ backend  (Spring Boot)
│
└─ 터미널에서 docker 명령어 실행
```

요청 흐름:

```jsx
[호스트 PC]  ──► [nginx :8000]  ──► [backend :8080]
   curl              80 → 8000          내부 전용
                  (호스트 노출)         (외부 차단)
```
- 클라이언트(호스트의 curl 또는 client 컨테이너)는 nginx만 알 수 있음
- 백엔드는 같은 Docker 네트워크 안에서만 접근 가능
- 이게 **리버스 프록시의 본질**: 클라이언트는 백엔드를 직접 만질 수 없음

## 3. Test 파일 만들기

### Backend Controller

```java
package com.example.backend;

import jakarta.servlet.http.HttpServletRequest;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.LinkedHashMap;
import java.util.Map;

@RestController
public class IpTestController {

    @GetMapping("/nginx")
    public Map<String, String> nginxTest(HttpServletRequest request) {
        Map<String, String> result = new LinkedHashMap<>();
        result.put("remoteAddr", request.getRemoteAddr());
        result.put("X-Forwarded-For", request.getHeader("X-Forwarded-For"));
        result.put("X-Real-IP", request.getHeader("X-Real-IP"));
        result.put("Host", request.getHeader("Host"));
        return result;
    }
}
```

### Dockerfile

```bash
FROM maven:3.9-eclipse-temurin-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn package -DskipTests

FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### nginx/nginx.conf

```bash
events {}

http {
    server {
        listen 80;

        location / {
            proxy_pass http://backend:8080;

            proxy_set_header Host              $host;
            proxy_set_header X-Real-IP         $remote_addr;
            proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

| 변수 | 의미 |
| --- | --- |
| `$host` | 클라이언트가 요청한 호스트명 |
| `$remote_addr` | nginx에 접속한 상대방 IP |
| `$proxy_add_x_forwarded_for` | 들어온 X-Forwarded-For 헤더 + `$remote_addr` 이어붙임 |
| `$scheme` | http / https |

### docker-compose.yml

```bash
services:
  backend:
    build: ./backend
    expose:
      - "8080"

  nginx:
    image: nginx:1.27-alpine
    ports:
      - "8000:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - backend

  client:
    image: curlimages/curl:latest
    command: sleep infinity
    depends_on:
      - nginx
```
- `expose: "8080"` — 같은 compose 내의 다른 컨테이너에만 노출. **호스트에서 접근 불가**.
- `ports: "8000:80"` — 호스트 PC의 8000 ↔ nginx 컨테이너의 80 연결.
- `volumes` — 로컬 nginx.conf를 컨테이너 안의 `/etc/nginx/nginx.conf` 위치에 마운트.
- `client`의 `sleep infinity` — curl만 깔린 컨테이너가 안 죽고 계속 떠있게.

## Test

### Test-1 (호스트 → nginx → backend )

```bash
curl http://localhost:8000/nginx
```

기대결과

```bash
{
  "remoteAddr": "172.x.x.x",       ← nginx 컨테이너 IP
  "X-Forwarded-For": "172.x.x.x",
  "X-Real-IP": "172.x.x.x",
  "Host": "localhost"
}
```
### Test-2 (호스트 → backend 직접)

```bash
curl http://localhost:8080/nginx
```

기대결과

```bash
curl: (7) Failed to connect to localhost port 8080
```

확인 포인트

- 백엔드는 `expose`만 했고 `ports` 안 했으므로 호스트에서 직접 접근 불가
- **리버스 프록시의 본질** — 클라이언트는 백엔드를 직접 만질 수 없음
- 실무 비유: WebtoB 뒤의 JEUS, 방화벽 안의 WAS

### Test-3 (client 컨테이너 → nginx → backend)

```bash
docker compose exec client curl http://nginx/nginx
```

기대결과

```bash
{
  "remoteAddr": "172.x.x.A",       ← nginx 컨테이너 IP
  "X-Forwarded-For": "172.x.x.B",  ← client 컨테이너 IP
  "X-Real-IP": "172.x.x.B",
  "Host": "nginx"
}
```

확인 포인트

- `remoteAddr`는 여전히 nginx
- `X-Forwarded-For`에 **진짜 클라이언트(client 컨테이너) IP**가 찍힘
- nginx 설정의 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`가 만들어준 것

### Test-4 (client 컨테이너 → backend 직접 (nginx 우회))

```bash
docker compose exec client curl http://backend:8080/nginx
```

기대결과

```bash
{
  "remoteAddr": "172.x.x.B",       ← client 컨테이너 IP가 직접
  "X-Forwarded-For": null,
  "X-Real-IP": null,
  "Host": "backend:8080"
}
```

확인 포인트

- nginx 안 거치니까 `remoteAddr`에 진짜 IP가 그대로 찍힘
- `X-Forwarded-For`는 `null` — 아무도 헤더를 안 박았으니까. **헤더는 자동 생성되지 않음**.
