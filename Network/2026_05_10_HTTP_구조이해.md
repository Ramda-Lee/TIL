# HTTP 구조 이해

**목표:** HTTP 요청/응답의 구조를 정확히 이해하고 각 구성요소의 역할을 설명할 수 있다.

요청 구조 이해 (요청 라인 / 헤더 / 빈 줄 / 바디)

```jsx
┌──────────────────────────────────────────┐
│ POST /file/upload HTTP/1.1               │ ← 요청 라인
├──────────────────────────────────────────┤
│ Host: api.bank.com                       │
│ Content-Type: application/json           │ ← 헤더
│ Content-Length: 47                       │
│ Authorization: Bearer eyJhbGc...         │
├──────────────────────────────────────────┤
│                                          │ ← 빈 줄 (헤더 끝 표시)
├──────────────────────────────────────────┤
│ {"fileId":"202505_abc","isDec":true}     │ ← 바디
└──────────────────────────────────────────┘
```

- 요청 라인: 메서드 / 경로 / 프로토콜 버전
- 헤더(Header)
    - name: value 형식의 메타데이터로 이루어짐
    - Host, Content-Type, Content-Length, Authrization 등
- 빈 줄: 헤더와 바디를 구분하는 경계선. \r\n\r\n (CRLF 두 번)
    - 이게 없으면 HTTP 파서가 어디까지가 헤더인지 모른다
- 바디(Body)
    - 실제 전송할 데이터. GET 요청은 보통 없고, POST/PUT/PATCH에 주로 포함

응답 구조 이해 (상태 라인 / 헤더 / 빈 줄 / 바디)

```jsx
┌──────────────────────────────────────────┐
│ HTTP/1.1 200 OK                          │ ← 상태 라인
├──────────────────────────────────────────┤
│ Content-Type: application/json           │
│ Content-Length: 124                      │ ← 헤더
│ Date: Thu, 08 May 2026 09:00:00 GMT      │
├──────────────────────────────────────────┤
│                                          │ ← 빈 줄
├──────────────────────────────────────────┤
│ {"status":"success","fileId":"202505..."} │ ← 바디
└──────────────────────────────────────────┘
```

- 응답 상태 라인: 프로토콜 / 코드 / 메시지
- 헤더 / 빈 줄 / 바디: 요청과 동일한 구조

주요 HTTP 메서드 정리 (GET / POST / PUT / PATCH / DELETE): 각 메서드는 “무엇을 하겠다” 의도

- GET: 조회
- POST: 생성/처리
- PUT: 전체 수정/생성
- PATCH: 부분 수정
- DELETE: 삭제

주요 상태코드 정리 (2xx / 3xx / 4xx / 5xx)

- 2xx: 성공
- 3xx: 리다이렉트
- 4xx: 클라이언트 에러
- 5xx: 서버 에러

주요 헤더 의미 이해 (Content-Type, Content-Length, Host 등)

- Content-Type: 본문이 어떤 형식인지 알려주는 헤더, charset도 같이 명시한다.
- Content-Length: 본문의 길이(byte)
- Host: 어떤 도메인을 향한 요청인지. HTTP/1.1 필수 헤더
- Authorization: 인증 정보
- Accrpt: 클라이언트가 받기 원하는 형식
- Cookie / Set-Cookie
    
    ```jsx
    [요청]  Cookie: SESSION=abc123; theme=dark
    [응답]  Set-Cookie: SESSION=xyz789; HttpOnly; Secure; Max-Age=3600
    ```
    
    - HttpOnly: JS에서 접근 불가
    - Secure: HTTPS에서만 전송
    - Samesite: CSRF 방어
- Cache-Control
    
    ```jsx
    Cache-Control: no-cache              ← 매번 검증
    Cache-Control: no-store              ← 절대 저장 X
    Cache-Control: max-age=3600          ← 1시간 캐싱
    Cache-Control: public, max-age=86400 ← 모두 캐싱 가능, 1일
    ```
    
- Content-Disposition: 파일 다운로드 시 핵심
    
    ```jsx
    Content-Disposition: attachment; filename="report.pdf"
    Content-Disposition: attachment; filename*=UTF-8''%ED%95%9C%EA%B8%80.pdf
    ```
    
    - attachment: 다운로드 처리
    - inline: 브라우저에서 바로 표시
    - filename*=’UTF-8’….: 한글 파일명 처리용
