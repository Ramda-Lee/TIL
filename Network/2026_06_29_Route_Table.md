# Route Table
목표: 서브넷의 성격을 실제로 결정하는 게 라우트 테이블이라는 걸 이해하고, 패킷이 목적지를 찾아가는 순서를 그릴 수 있다

라우트 테이블 

- 이 목적지로 가는 패킷은 저 타깃으로 보내라는 규칙
- 목적지 CIDR → 타깃(target) 매핑 규칙

```markdown
목적지 (Destination)      타깃 (Target)
──────────────────────────────────────
10.30.0.0/16             local           ← VPC 내부행
0.0.0.0/0                igw-0abc...      ← 그 외 전부(인터넷)행
172.16.0.0/12            pcx-0def...      ← Peering 상대 VPC행
10.50.0.0/16             vgw-0ghi...      ← 온프렘(전용선)행
```

- 서브넷에서 패킷이 나갈 때, AWS의 가상 라우터가 패킷의 목적지 IP를 이 표의 목적기 CIDR들과 대조해서 매칭되는 행의 타킷으로 보낸다.

**local 라우트** 

- 라우트 테이블을 새로 만들면 VPC CIDR을 향하는 local 경로가 자동으로 박혀있다. → 삭제 및 수정 불가능
- 서브넷끼리 별도 설정없이 통신이 되는 이유다.
- VPC 안에서는 서브넷이 다르고 AZ가 달라도 기본으로 다 통한다.

**Longest Prefix Match** 

- 목적지 IP가 표의 여러 행에 동시에 매칭이될 수 있다. → 더 구체적인(긴 프리픽스) 경로가 우선

```markdown
0.0.0.0/0     → igw-xxxx      (/0,  가장 넓음)
10.50.0.0/16  → vgw-xxxx      (/16, 더 구체적)

목적지 10.50.1.10 은 둘 다에 매칭되지만...
→ /16이 /0보다 길다(구체적) → vgw(전용선)로 보냄. IGW로 안 감.
```

`0.0.0.0/0`의 행선지가 IGW면 퍼블릭, NAT GW면 프라이빗

```markdown
0.0.0.0/0 → IGW (인터넷 게이트웨이)
   → 인스턴스가 인터넷과 직접 양방향 통신 (퍼블릭 IP 있으면 인바운드도 가능)
   → 이 라우트 테이블에 묶인 서브넷 = Public

0.0.0.0/0 → NAT Gateway
   → 내부에서 밖으로 나가는 건 되지만, 밖에서 안으로 먼저 못 들어옴
   → 이 라우트 테이블에 묶인 서브넷 = Private (단, 아웃바운드는 가능)

0.0.0.0/0 행이 아예 없음
   → 인터넷과 완전 단절 (VPC 내부 + 명시된 경로만 통신)
   → 가장 폐쇄적인 Private (DB 계층에 적합)
```

IGW와 NAT의 핵심 차이는 방향성이다. IGW는 양방향이라 웹서버에서 쓰이고 NAT는 단방향이라 내부 서버가 패치 받으러 나갈 때만 쓰인다.

Main 라우트 테이블 vs 커스텀 라우트 테이블, 서브넷 연결(association)

- VPC를 만들면 Main 라우트 테이블이 하나 자동 생성된다. → 어떤 라우트 테이블에도 명시적으로 연결이 되지 않은 서브넷은 자동으로 Main 테이블을 따른다.
- 주의사항: Main 테이블에 0.0.0.0/0 → IGW를 넣어두면 연결을 깜빡한 서브넷이 전부 의도치 않게 퍼블릭이 되버린다. → DB 서브넷이 인터넷에 노출되는 사고
- 권장 패턴은 Main 테이블은 가장 안전하게(IGW 경로 없이)두고, 퍼블릭으로 만들 서브넷만 IGW가 붙은 커스텀 테이블에 명시적으로 연결한다.
- 연결은 서브넷 1개 → 라우트 테이블 1개 관계이다. → 한 서브넷이 동시에 두 테이블을 따를 순 없다. 반대로 한 라우트 테이블에는 여러 서브넷을 연결할 수 있다.

핵심 질문: "겉보기엔 똑같이 생긴 서브넷 두 개인데 하나는 인터넷이 되고 하나는 안 된다. 어디를 봐야 하는가?"

- 그 서브넷에 연결된 라우트 테이블의 0.0.0.0/0행을 찾아본다
- 디버깅 순서
    1. 이 서브넷이 어느 라우트 테이블에 연관 되어 있나
    2. 그 테이블에 0.0.0.0/0이 있는지
    3. 있다면 타깃이 IGW인가 NAT인가
